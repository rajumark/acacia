# Acacia DSL Function Mappings

This document provides complete mapping of all Acacia DSL functions with examples for AI model training.

## Format
Each function mapping follows this structure:
- Original Function: Full Modifier function name
- Short Function: Concise Acacia DSL name  
- Before: Verbose Compose code
- After: Clean Acacia DSL code

## Categories

### Layout Functions
Functions related to positioning, sizing, and layout management.

### Styling Functions  
Functions for visual appearance including background, borders, and effects.

### Interaction Functions
Functions for user input handling including clicks, drags, and scrolling.

### Padding Functions
Functions for spacing and padding management.

### Size Functions
Functions for width, height, and size constraints.

### Effects Functions
Functions for visual transformations like blur, brightness, and color filters.

### Other Functions
Additional utility functions including semantics, layout IDs, and system bar handling.

## Usage Examples

### Basic Layout
```kotlin
// Before
Modifier.fillMaxWidth().fillMaxHeight().padding(16.dp)

// After  
Modifier.fmw().fmh().p(16.dp)
```

### Styling
```kotlin
// Before
Modifier.background(Color.Blue).shadow(
    elevation = 4.dp,
    shape = RoundedCornerShape(8.dp)
)

// After
Modifier.bg(Color.Blue).sh(4.dp).cp(8.dp)
```

### Interaction
```kotlin
// Before
Modifier.clickable(
    onClick = { /* handle click */ }
).scrollable(
    state = rememberScrollState()
)

// After
Modifier.clk { /* click */ }.sc2(state)
```
